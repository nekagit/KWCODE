# Prompt: User guide

Use when writing or updating user-facing documentation.

## Context

- App: (what the app does; see .cursor/project/PROJECT-INFO.md).
- Audience: end users or internal users.

## Instructions

1. Structure by user tasks (getting started, main flows, troubleshooting).
2. Use clear headings and steps; add screenshots or diagrams if helpful.
3. Write to `docs/guides/` or `.cursor/documentation/` as appropriate.
4. Link to setup and API docs where relevant.

## Output

- User guide sections (markdown).
