## Prioritized work items (tickets)

Use checklist lines so the Kanban and Run tab can parse them. Format:

- Under a priority heading: `### P0` … `### P3`
- One ticket per line: `- [ ] #N Title — optional description — @agent1 @agent2` (use `[x]` when done)
- Optional: `#### Feature: Name` to group tickets under a feature

Example (uncomment or edit to add real work):

### P1 — High / quality and maintainability

- [ ] #1 Example ticket — Add real tickets in the Kanban, then Sync to persist here.
