# Prompt: Technical architecture documentation

Use when writing or updating technical architecture docs.

## Context

- Architecture: `.cursor/setup/architecture.md`, `.cursor/setup/frontend.json`, `.cursor/setup/backend.json`.
- ADRs: `.cursor/adr/`.

## Instructions

1. Describe layers, data flow, and key design decisions.
2. Reference ADRs for major decisions.
3. Update `.cursor/documentation/architecture-overview.md` or `docs/architecture/`.
4. Use Mermaid for diagrams if helpful.

## Output

- Architecture doc (markdown).
